PRD - Sistema de Ventas de Calzado
1. Descripción del Proyecto
La tienda de calzado registra ventas manualmente, causando desorden y pérdida de información.

2. Objetivo
Crear un sistema web para controlar ventas, clientes y productos.

3. Roles
Administrador
Vendedor
Cajero
4. Requisitos Funcionales
Registrar ventas
Registrar clientes
Registrar productos
Buscar ventas
Calcular ganancias
Registrar pagos fiados
5. Requisitos No Funcionales
Fácil de usar
Seguro
Rápido
Diseño ordenado
6. Casos de Uso
Venta al contado
Cliente compra botas y paga completo.

Venta fiada
Cliente compra zapatos y paga después.

7. Datos principales
Producto
Cantidad
Precio
Cliente
Tipo de pago
8. Criterios de Éxito
Registrar ventas sin errores
Consultar clientes
Calcular ganancias correctamente
